**Risk Assessment (post v1.2.15):** Both remaining items are **low criticality**, even on large sites (1M+ orders, 360k coupons):
- The N+1 fallback was the real risk — now fixed (2 batch queries, LIMITed).
- Worst case without backfill: first searches hit `LIKE '%term%'` on `wp_posts` (full scan but LIMITed) + 1 batch meta query. Not fast, but won't overload the DB.
- Lazy backfill self-heals at 10 coupons/search. After a few dozen searches, FULLTEXT lookup handles common queries.
- Order count is irrelevant — these TODOs only affect coupon search. Coupon count is what matters.
- README docs are polish; ops teams deploying on 1M-order sites will find `wp kiss-woo --help` or the diagnostic page.
- Admin notice is UX polish, not a safety or performance concern.
