## 4. Recent Lessons Learned
**Capture insights to improve processes and avoid repeating mistakes.**

1. **FULLTEXT > LIKE for multi-column search**: `OR title LIKE '%term%'` forces MySQL to abandon ALL indexes. FULLTEXT with BOOLEAN MODE gives sub-ms search on 360k+ rows.
2. **Always verify CHANGELOG claims against code**: v1.2.8 claimed error_log removal complete but 7 calls remained; v1.2.2 claimed debug_log removal but 9 call sites persisted.
3. **N+1 queries hide behind "temporary" fallback paths**: The `new WC_Coupon()` loop was "only for fresh installs" but was the hot path until backfill completed. Batch loading is always worth the effort.
4. **`$wpdb->esc_like()` is wrong for FULLTEXT**: It escapes `%` and `_` (LIKE operators), not `+`, `-`, `~`, `<`, `>` (FULLTEXT boolean operators). Different query types need different sanitization.

## Bonus: Project Scope Outline

> **Note:** This section is a work-in-progress and is being developed as a natural extension of the 4x4 methodology. It is not yet a core part of the framework but is included here as a supplementary tool for teams who want to add more context to their planning.

### 1. Goals
- Provide sub-second admin search across customers, orders, and coupons on WooCommerce stores with 1M+ orders and 360k+ coupons.
- Support WooCommerce HPOS (High-Performance Order Storage) and legacy `wp_posts` storage.
- Compatible with Smart Coupons and Advanced Coupons plugins.

### 2. Assumptions
- MySQL 5.6+ available (required for InnoDB FULLTEXT indexes).
- Sites with 100k+ coupons will run `wp kiss-woo coupons backfill` before enabling coupon search.
- Admin users have `manage_woocommerce` capability.

### 3. Potential Risks
- **Large-scale FULLTEXT performance untested** — Need to validate with 100k+ real coupon data (backlog item #4).
- **Guest order reassignment edge case** — Orders can become invisible if billing email differs from customer account email (backlog item #1).
- **Lazy backfill on cold start** — First searches on unpopulated sites hit `LIKE '%term%'` fallback on `wp_posts`. Mitigated by LIMIT cap and self-healing backfill.

### 4. Long-Term Maintainability
- Centralized debug logging via `KISS_Woo_Debug_Tracer` — all logging respects `KISS_WOO_FAST_SEARCH_DEBUG` constant.
- Single write path for coupon indexing (`KISS_Woo_Coupon_Lookup::upsert_coupon()`).
- Shared batch processor (`KISS_Woo_Coupon_Lookup_Builder`) used by both WP-CLI and admin UI background jobs.
- Lookup table schema versioned (`DB_VERSION = '1.1'`) with auto-upgrade via `dbDelta()`.
